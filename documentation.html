Project Objective

The programmatic purpose of the classification engine.

The primary objective of **FuturePath AI** is to predict the most suitable career trajectory for students by analyzing their academic indicators (CGPA), analytical logical aptitude levels, coding and software construction capabilities, communication skills, and primary domain interests. Using a pure Python custom decision tree classifier, it computes affinity ratings and provides structured career path roadmaps.

Methodology & Execution Pipeline

Step-by-step description of the data loading, training, evaluation, and prediction pipeline.

1
Pipeline Step 1 Dataset Loading

1. Dataset Loading

The model ingest pipeline reads student records from a formatted CSV dataset containing attributes: CGPA, CodingSkill, CommunicationSkill, AptitudeLevel, InterestArea, and Target CareerPath.

2
Pipeline Step 2 Data Cleaning

2. Data Cleaning

String values are trimmed and checked for structural validity. Missing data or invalid entries are systematically sanitized or skipped to prevent computation errors during classification.

3
Pipeline Step 3 Data Encoding

3. Data Encoding

Categorical string features (e.g., skill ratings and interest domains) are mapped to numeric values using standard dictionaries so they can be processed by the quantitative Decision Tree algorithms.

4
Pipeline Step 4 Train-Test Split (80/20)

4. Train-Test Split (80/20)

The dataset is shuffled randomly with a set seed for reproducibility. It is then partitioned: 80% is allocated for model training, and 20% is reserved for post-training out-of-sample accuracy evaluation.

5
Pipeline Step 5 Model Training

5. Model Training

A custom Decision Tree Classifier is fitted on the training split, recursively identifying optimal features and Gini impurity splits to construct decision nodes down to a max depth of 5.

6
Pipeline Step 6 Prediction

6. Prediction

The trained model traverses its generated decision tree paths based on a student profile input query, outputting a specific career classification label (e.g., AI Engineer or Cloud Engineer).

7
Pipeline Step 7 Accuracy Evaluation

7. Accuracy Evaluation

Model predictions are compared directly against the true labels of the 20% validation subset, yielding a quantitative accuracy index score matching the theoretical design performance.
